How to use Paper Networks Audit

Find groups of papers that share suspicious connection attributes (IP, email, device fingerprint, Word metadata, and more) to support papermill investigation.

Quick start

  • Choose connection types, apply filters, then click a cluster in the list or graph.
  • Open Compare papers to review the cluster side by side; click a paper node for the detail panel.
  • Set your name under Reviewer so seen, checked, resolve, and flag actions are attributed to you.

Features

Connection types & clustering
  • Toggle attributes: email, IP, device fingerprint, locale, WD author / last edited / company, author-declared IP.
  • Single-type clusters — papers sharing one value (e.g. same hashed IP).
  • Multi-type (AND) clusters — papers sharing the same value on every selected type; shown in lime on the graph.
  • Email regex — filter to matching author emails; combines with other types as AND, or lists matches alone.
  • Cluster pills show paper count, distinct authors/orgs, and metadata (journal, status, etc.).
Filters
  • Different submitting authors / organisations — exclude homogeneous clusters.
  • Only flagged clusters, hide all-rejected clusters, low name/email similarity.
  • Journal and status (rejected only / active only).
  • Min papers and sort by size, distinct authors, or distinct orgs.
  • List header shows cluster count and unique papers across those filtered clusters (a paper in several clusters is counted once).
  • Search — paste article IDs (comma, tab, or line separated) to narrow the data source to those papers; then connection types cluster within that set. With no connection type selected, all search papers appear. Or type a connection value to jump to it.
Graph
  • Overview — top clusters by size; double-click a hub to focus.
  • Focus — one cluster expanded: hub(s) for shared values, papers as nodes, edges for membership.
  • Paper size = papermill score band (grey → green → orange → red).
  • Dashed lines between papers = two or more shared attributes beyond the focused cluster.
  • Red outline = flagged; cyan ring / 👁 on hub = all papers in cluster seen.
  • Re-layout re-runs the force layout; hover nodes for tooltips.
Rejection reasons
  • Rejected papers include a rejection reason from the editorial system (e.g. Ethical issues, Below Standard, Author request).
  • In the compare table, the status column uses short codes (REJ) with colour by risk: red high (ethical issues, rejected before review), orange mid (below standard, prescreening, out of scope, not sufficiently revised, other), grey low (author or editorial withdrawal).
  • Hover the status cell for the full reason (e.g. “Rejected · Ethical issues”).
  • The detail panel shows status and a separate rejection-reason row when recorded.
  • Withdrawn papers appear as Rejected with reason Author request or EOF withdrawn — not a separate stage.
Detail panel
  • Paper metadata, papermill score, rejection reason (when rejected), connections to other papers in the cluster.
  • Click shared connection rows to focus that sub-cluster on the graph.
  • Flag as manufactured with optional comment; resolves and flags are mutually exclusive.
  • Article IDs link to Frontiers review; opening a link records checked.
Compare table
  • All papers in the current cluster; shared values highlighted in connection-type colours.
  • Sort by any column; drag headers to reorder (saved in this browser).
  • Shift + scroll wheel or ← → for horizontal scroll; hover for full cell/header text.
  • Status column: risk colour on rejected rows; hover for the full rejection reason.
  • Copy exports the visible table as TSV for spreadsheets.
  • Rejected rows sort to the bottom when sorting by papermill score.
Dataset field filter
  • Select a row, then click a cell to filter the entire snapshot by that field value.
  • Filter bar appears in Filters; clear it to return to the full dataset.
  • Compare subtitle shows match count across the snapshot.
Team tracking
  • 👁 Seen — auto on row scroll, detail view, or graph click (unless incognito).
  • ↗ Checked — review link opened from compare or detail.
  • ✓ Resolved — marked non-problematic; clears flag.
  • ⚑ Flagged — marked manufactured; clears resolve.
  • Click any icon for full history (user + timestamp). Hover for a quick list.
  • Bulk: resolve, flag, unflag, clear resolve, mark unseen, mark unchecked.
  • Cluster 👁 when every paper has a seen record; lists who viewed what.
Flags & exports
  • ⚑ toolbar — list all flagged papers in the snapshot.
  • ↓ CSV — all flag records (active and removed): id, flagger, comment, timestamps.
  • ⟳ reloads flags and workflow data from the server.
Manager summary
  • Password-protected panel under Reviewer (remembered in this browser).
  • Per-user counts: seen, checked, resolved, flagged articles.
URL sharing
  • URL updates live: connection types, filters, focused cluster, selected article, compare panel.
  • Bookmark or share the link to restore the same view for a colleague.
